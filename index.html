Overview

The Crisis &
Our Solution

⚠ National Agricultural Emergency

Sri Lanka's livestock sector faces a severe dual crisis: feed costs have risen over 355% since 2019, and Cyclone Ditwah in 2025 devastated more than 108,000 hectares of paddy rice, severing the supply chain for animal feed entirely.

In this context, hydroponic fodder production (the soilless cultivation of sprouted grains) emerges not just as an alternative, but as a necessity. It offers a land- and water-efficient method to produce high-quality green feed within controlled indoor environments, completely decoupled from soil quality and seasonal rainfall.

This project presents an integrated system using IoT hardware, deep learning, and machine learning. A unique scientific contribution is the Poaceae Transfer Learning hypothesis: exploiting botanical taxonomic similarity between rice, maize, and sorghum to train disease detection models on large rice datasets and transfer them to fodder crops.

The system achieved a peak mAP@0.5 of 0.843 in disease detection and validated reliable sensor telemetry at 2-second cadence with hardware crash-detection across extended trials.

Intelligent Modules

Four AI Components

Each module addresses a distinct challenge, from assessing flood-damaged seeds to minimising energy waste in post-disaster conditions.

01
ResNet-50 · Multi-Task Learning
Seed Identification & Quality Analysis
A multi-task transfer-learning architecture based on ResNet-50 performs simultaneous rice type classification and quality analysis from a single RGB image at 384×384px. Enables safe use of flood-damaged seed stock without expensive hyperspectral hardware.
>92%Classification Accuracy Target
02
YOLOv11s · Poaceae Transfer Learning
Multi-Modal Disease Detection
Fuses YOLOv11s visual features with real-time IoT sensor telemetry to detect 7 pathological conditions. Trained on 19,000 rice leaf images and transferred to maize and sorghum via the novel Poaceae Transfer Learning hypothesis. Trained for 600 epochs.
0.843Peak mAP@0.5 Achieved
03
YOLOv11s · C2PSA Attention · ESP32-CAM
Plant Growth Stage Detection
Identifies four key growth phases: Seed, Germination, Sprout, and Seedling, from live ESP32-CAM footage using C2PSA spatial attention. Triggers phase-specific actuator adjustments and harvest-window alerts. The optimal protein window spans only 12–18 hours in a 7-day growing cycle.
80–88%Overall Accuracy Expected
04
LSTM · PZEM-004T · Threshold Control
Energy Prediction & Actuator Control
A stacked LSTM model (64→32 units) trained on live PZEM-004T sensor telemetry predicts energy demand over 24-timestep sliding windows. Threshold-based scheduling staggers pump activation to cut peak current draw across complete growing cycles.
18–25%Target Energy Reduction

Sensor Input Layer
A heterogeneous sensor network collects environmental telemetry at 2-second cadence. The ESP32 Manager handles stationary sensor polling and actuator control; the ESP32-CAM on a motorized CNC gantry rail scans crops continuously.
DHT22 Temp/HumiditypH ProbeEC SensorCO₂ SensorPZEM-004T PowerESP32-CAMWater LevelSoil Moisture
Communication Layer
Secure MQTT protocol streams sensor telemetry in real time. HTTP POST carries image frames from ESP32-CAM to the FastAPI backend. All communications are TLS-encrypted over the local 2.4GHz Wi-Fi star network.
MQTTHTTP POSTTLS EncryptionWebSocketWi-Fi 2.4GHzService Discovery
Cloud Backend Layer
The Python FastAPI server hosts all four AI modules. Hardware safe-mode crash detection and atomic I²C relay de-energization are implemented at firmware level. Safety mechanisms absent from most comparable research prototypes.
Python FastAPIYOLOv11sResNet-50Stacked LSTMLabel StudioHardware Crashdetect
Actuator Control Layer
Relay-controlled actuators execute AI commands dispatched as MQTT JSON payloads. The ESP32 firmware switches relays through an optocoupler board. Staggered pump activation reduces instantaneous current draw during high-load periods.

User Interface Layer
A React web and React Native mobile application provides real-time sensor dashboards, AI-generated disease alerts, growth stage notifications with harvest-window countdowns, energy analytics panels, and manual actuator override controls.
